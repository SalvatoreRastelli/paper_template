\begin{abstract}
	Centralized learning achieves the best statistical performance by
	aggregating all observations, but such global coordination is unavailable
	in decentralized cooperative systems.
	We propose \merw, a decentralized communication protocol on undirected
	graphs that establishes, from purely local interactions, a coordinator
	among the agents.
	\merw uses eigenvector centrality to 1) choose the most central node as
	coordinator, and 2) construct a routing tree, rooted at the coordinator
	node, which spans the graph.
	Information flows according to this routing tree: from leaves to root
	during the aggregation phase, and vice versa during the broadcasting phase.
	We prove that, under this protocol, a bandit algorithm based on the
	coordinator's information state achieves the same statistical performance
	as a centralized instance, at the cost of $O(D)$ communication rounds, with
	$D$ the tree's diameter.
	The protocol can be used in various decentralized cooperative learning
	setups, of which we show two---regret minimization and best arm
	identification---where the method recovers centralized-like results in
	finite time.
	We further present a fault-tolerant version of the protocol, addressing the
	coordinator's structural weakness as a single point of failure: any node,
	including the coordinator, can fail and be replaced by re-election.
\end{abstract}

\section{Introduction}

In cooperative multi-agent learning, no individual agent has access to the
global state of the system, but the best-performing strategy is usually the one
available to a centralized decision maker with a full view of everyone's
observations. This is the setting whenever $N$ agents connected by an
undirected graph $G = (V, E)$ must repeatedly act and learn from feedback while
exchanging only local messages with their graph neighbors. Existing approaches
close the gap to centralized performance either by symmetric averaging, where
every agent maintains its own estimate and no coordinator ever exists (e.g.,
Coop-UCB2 \citep{landgren2021distributed}), or by assuming a communication
model rich enough that any agent can reach any other in one step (e.g., the
all-to-all broadcast of \cite{hillel2013distributed}). 

We instead ask whether the communication graph $G$ itself can select which
agents should play a coordinating role. This motivates measuring an agent's
importance through a global notion of centrality. Building on this idea, we
propose \merw, a method to \emph{construct}, in a completely decentralized
way, a communication structure according to the eigenvector centrality induced
by $G$.

In \merw, a coordinator \emph{emerges} from the graph structure, identified
from purely local computation, with no agent ever seeing the global topology.
The key idea is that eigenvector centrality, computed
via a fully decentralized gossip protocol \citep{jelasity2007asynchronous},
can be used to identify a ``most central'' node in $G$. One can then construct
a routing tree which spans $G$ via max-flooding, again only using local
information. Once the tree is in place, the hub acts as a de-facto central
coordinator: every agent routes its observations up toward the hub, which sees
the globally aggregated picture and broadcasts its decisions back to the other
nodes. We prove that this emergent coordinator is statistically equivalent to a
true centralized coordinator for any decision rule that depends only on
globally aggregated per-agent statistics, at the cost of a bounded
communication overhead determined by the tree's depth.

Any decentralized cooperative-learning task whose decision rule depends only
on globally aggregated per-agent statistics can therefore run over the same
emergent hub-and-tree structure, by supplying two application-specific rules:
1) what each agent contributes locally, and 2) what the hub decides from the
aggregate. \merw is this protocol, parameterized by those two rules; we
instantiate both for two standard
cooperative multi-armed bandit objectives, \emph{regret minimization} and
\emph{best arm identification} (BAI), and show in both cases that the
emergent coordinator recovers centralized-level performance.

The emergent-coordinator design has one structural weakness: the hub is a
single point of failure, and more generally, any node's failure disrupts the
part of the tree that routes through it. We handle every failure with one
procedure: a dead node is a low-rank perturbation of the graph, the survivors
re-run the same eigenvector computation that elected the original hub on the
perturbed graph, warm-started from the pre-failure centralities, and rebuild the
tree around whoever is now most central. No node is treated specially and no
backup is kept. Removing the hub is simply the largest perturbation, so it is
where re-election has the most work to do; a low-degree node barely moves the
centralities and re-election reduces to a local re-attach. The cost is governed
by the graph's spectral gap, the same quantity that governs hub emergence in
Section~\ref{sec:mdst-gap}.

\paragraph{Contributions.} (1) A fully decentralized protocol for building a
centralized communication structure: the hub and tree emerge from local
computations, requiring no privileged node, no global communication, and no
prior knowledge of the graph structure. (2) A proof that this emergent
coordinator is statistically sufficient for any decision rule depending only on
globally aggregated statistics, instantiated for regret minimization and best
arm identification, both matching centralized performance up to an $O(D)$
communication overhead per round, where $D$ is the depth of the induced tree.
(3) \merwft, a spectral re-election extension tolerating permanent failure of any
node, hub or non-hub, through a single procedure whose cost is set by the graph's
spectral gap.

\section{Related Work}
\label{sec:related}

\paragraph{Cooperative multi-agent bandits.} Coop-UCB2
\citep{landgren2021distributed} diffuses observations across the graph with
doubly stochastic consensus weights; every agent plays an equivalent role and
no coordinator is assumed. \cite{martinez2019decentralized} introduce the
total-pull clock we adopt to index UCB confidence intervals by actual
observations rather than wall-clock rounds. \cite{hillel2013distributed} study
cooperative BAI under all-to-all broadcast; we build directly on their
Algorithm 3, replacing the broadcast with hop-by-hop relay over the routing
tree. All three fix the communication model in advance, symmetric gossip or
all-to-all; \merw instead derives the communication structure, the hub and
tree, from the graph itself, and is sufficient for the same class of decision
rules.

\paragraph{Eigenvector centrality and decentralized computation.}
Eigenvector centrality appears as the stationary distribution of Maximal
Entropy Random Walk \citep{burda2009localization}, which is the
information-theoretic argument we use to justify it (Sec.~\ref{sec:merw}).
Computing it exactly requires solving the graph's eigenvalue problem;
\cite{jelasity2007asynchronous} give a gossip-based normalization for
distributed power iteration that avoids a global all-reduce, which we use
directly, with both the fixed iteration count and the gossip-round count set
to $O(\log N)$.

\paragraph{Leader election and fault tolerance.} Classical leader election,
Bully \citep{garciamolina1982} and ring-based extrema-finding
\citep{changroberts1979}, elects the highest-ID surviving process by
message-passing; the result depends only on process identities, not on graph
structure, so a failure is repaired by the same election regardless of which
node is lost or how central it was. \merwft elects by eigenvector centrality
instead, so a failure is a change to the quantity being elected on, not just a
removal from the candidate pool: we treat it as a low-rank perturbation of the
adjacency matrix and bound the resulting movement of the leading eigenvector by
the Davis--Kahan theorem \citep{daviskahan1970}. Perturbation analysis of
graph spectra under node removal has been studied for how it shifts a
network's dynamical behavior \citep{restrepo2006}; we use the same perturbation
view constructively, to warm-start the re-election itself rather than to
characterize the shift. This ties the repair cost to the spectral gap and lets
one procedure cover any node's failure, hub or not, with no backup and no
special case.

\paragraph{Minimum-diameter spanning trees.} Choosing the coordinator to
directly minimize the routing tree's diameter is a solved problem: the optimal
root is the graph's absolute center, and a shortest-path tree rooted there is a
minimum diameter spanning tree (MDST) \citep{hakimi1964,hassintamir1991}. This
has been made fully distributed
\citep{bui2013distributed,butelle1995selfstabilizing}, including a
self-stabilizing version that recovers after a fault, but the cost is
structural: computing the absolute center requires every node to hold
all-pairs shortest-path information, $O(N)$ per-node distance entries at
minimum and $O(N^2)$ total network state \citep{bui2013distributed}, not just
its own neighborhood. A node failure changes the graph itself, so the center
may move, and even the self-stabilizing variant reacts by reconverging over
the whole network rather than localizing the repair to the failure.
\merw only ever stores distances to immediate neighbors, and \merwft recovers
from a single node failure by a warm-started re-election costing
$O(\log(N/\epsilon))$ rounds on an expander, without touching the rest of the
network.

\section{EigenTree: The Protocol}

\subsection{Hub Emergence and Routing}
\merw is based on the eigenvector centrality; we describe here how we obtain
it in a purely decentralized way, and how we use it to construct a routing tree
that spans the graph $G$.

Formally, let $A \in \{0,1\}^{N \times N}$ be the adjacency matrix of $G$. By
the Perron--Frobenius theorem \citep{hornjohnson2013}, $A$ has a unique
positive leading eigenvalue $\lambda_1$ with strictly positive eigenvector
$\psi$ such that $A\psi = \lambda_1 \psi$ and $\|\psi\|_2 = 1$.

No single agent has access to $A$, so $\psi$ is computed by the distributed
power iteration of \citep{jelasity2007asynchronous}. Each agent $i$ keeps a
local scalar $w_i$, initialized to 1, and repeats: (1) collect $w_j$ from every
neighbor and form the unnormalized update $\tilde w_i = \sum_{j\sim i} w_j$;
(2) compute the local log-growth rate $r_i = \log(\tilde w_i/w_i)$; (3) average
$r_i$ with random neighbors for $O(\log N)$ gossip rounds, so that every $r_i$
converges to the same network-wide estimate of $\log\lambda_1$ without any node
ever summing over all of $N$; (4) normalize locally, $w_i \leftarrow \tilde
w_i/\exp(r_i)$. This runs for a fixed number of iterations $\tau_{\mathrm{init}}
= O(\log N)$, so all agents finish on the same round and begin max-flooding
together, with no agent evaluating a global convergence test. This gossip step
replaces the single global $\ell_2$-norm renormalization an ordinary
(non-distributed) power iteration would need, at the cost of the extra gossip
rounds. Importantly, only the \emph{ratio} $\psi_j/\psi_i$ is needed for
routing: agent $i$ simply checks which neighbor's value is larger. Thus, the
common unknown scale $\tilde\psi_i \approx c\cdot\psi_i$ left behind by this
normalization is never resolved and never needs to be: no agent ever learns
$c$.

After power iteration, a short \emph{max-flooding} phase merges any local
maxima of $\tilde\psi$ into a single spanning tree. Each agent $i$ keeps a
flooded value $m_i$, initialized to $\tilde\psi_i$, and a pointer
$\mathrm{via}_i$ initialized to itself. Every round, $i$ broadcasts $m_i$ to
its neighbors; if some neighbor $j$ reports $m_j > m_i$, $i$ sets $m_i
\leftarrow m_j$ and $\mathrm{via}_i \leftarrow j$. Every node runs this for a
fixed number of rounds $T_{\mathrm{flood}}$, a protocol hyperparameter set to
any value at least $\mathrm{diam}(G)+1$; no node needs to know $\mathrm{diam}(G)$
itself, only this common round budget, and because the budget is shared and the
rounds are synchronous, all nodes finish flooding on the same round and begin
the first pull together. Since flooding propagates the global maximum one hop
per round, $\mathrm{diam}(G)+1$ rounds suffice for it to reach every node. At
convergence, $m_i$ equals the global maximum $\tilde\psi_{h^\star}$ everywhere,
and $\mathrm{via}_i$ points one step closer to $h^\star$; any node whose own
$\tilde\psi_i$ was a local but not the global maximum re-routes toward
$\mathrm{via}_i$, while $h^\star$ itself, whose value never has a better
neighbor, keeps no parent and becomes the root. The result is a single directed
spanning tree $\mathcal{T}$ rooted at $h^\star = \arg\max_i \tilde\psi_i$,
formed within the $T_{\mathrm{flood}}$ flooding rounds. Immediately afterward, each
node $i$ knows only its own parent $\mathrm{par}(i)$; its children, its depth
$d_i$, and the tree height $D$ are all discovered incrementally during the
protocol's first cycle below, not known in advance.

Complete proofs of every theorem, proposition, and corollary in this paper are
given in the supplementary material, using matching notation and statement
numbering; we give the proof of Theorem~\ref{thm:coordinator} here as well,
since it is the paper's central claim.

\begin{proposition}[Centralized aggregation and broadcast via routing]
\label{prop:aggbcast}
If each node $i$ holds a scalar $x_i$ and propagates it toward $h^\star$ one
	hop per round (each intermediate node forwarding the sum received), the hub
	holds $\sum_i x_i$ after $D$ rounds, where $D$ is the tree height.
	Symmetrically, a value broadcast from $h^\star$ one hop per round reaches
	every node after $D$ rounds. Neither direction requires any node to know
	$D$ in advance: the uplink terminates when the hub observes a round with no
	incoming packets, and each node infers its own depth $d_i$ from the hop
	count carried on the first downlink it receives.
\end{proposition}

\subsection{A Path-Based Motivation for \merw}
\label{sec:merw}

We provide here a heuristic for choosing eigenvector centrality as our guiding metric.

In a federated setup, nodes share their states and perform updates. One can
think of the state information to be shared as a ``packet'' that undergoes a
random walk on the graph. Let us consider two nodes $i,j \in G$, with a packet
originating from $i$ and reaching $j$. For a path length $L \geq
\mathrm{dist}_G(i, j)$, there are generally many distinct paths joining $i$ and
$j$. We assume there is no reason for one specific path to be favored compared
to another: all admissible paths of a certain length $L$ which connect $i$ and
$j$ are thus equally likely. Crucially, this condition is \emph{non-local};
while an unbiased random walk prescribes all neighbors to be equally likely,
this process considers all long paths. The question then becomes: which node
$h\in G$ is most frequently visited under this path ensemble? That node will be
involved in the largest portion of message exchanges, and thus be a natural
candidate to be a hub.

The framing we just described is a Maximal Entropy Random Walk (MERW) on $G$
\citep{burda2009localization}. The MERW is the walk on $G$ whose transition
probabilities make every path of a given length between two fixed nodes equally
likely, unlike the standard random walk, which only weighs each single step by
degree and has no notion of the paths it is part of. Because MERW weighs entire
paths equally, a node's stationary probability under MERW reflects its
participation in long paths throughout the graph, not just how many neighbors
it has.

MERW's stationary distribution is proportional to the eigenvector centrality,
and this is the link between MERW and \merw. Computing the eigenvector
centrality allows one to identify the hub node $h\in G$ which is most
frequently traversed.

\subsection{Coordinator Sufficiency}

\begin{theorem}[Coordinator sufficiency]
\label{thm:coordinator}
Let $\mathcal{A}$ be any decision rule whose output at each step depends only on globally aggregated per-agent statistics $B = \sum_{i \in V} b_i$, where $b_i$ is agent $i$'s local contribution. If a coordinator receives the exact aggregate $B$ within $D$ rounds and broadcasts its decision within $D$ further rounds, with all agents synchronized via the depth signal carried on the broadcast, then the coordinator's information state at every decision point is identical to that of a centralized instance of $\mathcal{A}$, with an additive overhead of at most $2D$ time steps per decision round.
\end{theorem}

\begin{proof}
Fix any decision round $r$. By Proposition~\ref{prop:aggbcast}, after $D$ rounds the hub holds $B = \sum_{i\in V} b_i$ exactly, not an approximation, since every node's contribution is included by the uplink termination argument. Because $\mathcal{A}$'s decision at round $r$ depends only on $B$, the hub computes exactly the action a centralized instance of $\mathcal{A}$ would compute given the same per-agent contributions. By Proposition~\ref{prop:aggbcast} again, within a further $D$ rounds every agent holds the hub's decision and has derived its own depth $d_i$ from the hop count on this broadcast, so all agents can identify when the current cycle ends and begin the next round's contribution simultaneously, regardless of their depth in $\mathcal{T}$. Hence the sequence of aggregates and decisions seen across the whole run is identical, round for round, to what a centralized instance of $\mathcal{A}$ would see given the same sequence of per-agent contributions; the only difference is the $2D$ rounds of communication latency inserted between successive decisions.
\end{proof}

\merw runs in synchronous rounds. Every agent takes one action per discrete
time step. A pull, a forwarded packet, any outcome of a step, becomes visible
to its recipient only at the start of the next step.

Algorithm~\ref{alg:merw-generic} gives \merw as a single generic protocol.
After initialization, agents repeat a fixed cycle of four phases: Pull,
Uplink, Hub decision, Downlink. An application supplies $\textsc{Init}$,
$\textsc{LocalPull}$, and $\textsc{HubDecide}$, which fix what each agent
contributes locally and what the hub decides from the aggregate $B$; Uplink
and Downlink carry whatever these three produce. Any decentralized learning
task whose decision rule fits Theorem~\ref{thm:coordinator} can run over the
same emergent tree this way, just by choosing a different $\textsc{Init}$,
$\textsc{LocalPull}$, and $\textsc{HubDecide}$. Table~\ref{tab:instantiations}
gives the full cycle for regret minimization and best arm identification,
detailed in the two subsections below.

\begin{algorithm}[tb]
\caption{\merw (generic protocol)}
\label{alg:merw-generic}
\begin{algorithmic}[1]
\STATE \textbf{Require:} graph $G$; hooks $\textsc{Init}$, $\textsc{LocalPull}$, $\textsc{HubDecide}$; a stopping condition on $t$ or on a hub-computed flag $\mathrm{done}$
\STATE Compute $\tilde\psi$ via distributed power iteration; run max-flooding to obtain tree $\mathcal{T}$ rooted at $h^\star$
\STATE $t \leftarrow 0$; $\mathrm{done} \leftarrow \textbf{false}$; each node $i$ knows only $\mathrm{par}(i)$ so far
\FORALL{$i \in V$}
    \STATE $\sigma_i \leftarrow \textsc{Init}()$; $t \mathrel{+}= (\text{rounds used})$
\ENDFOR
\REPEAT
    \FORALL{$i \in V$}
        \STATE $(b_i, \sigma_i) \leftarrow \textsc{LocalPull}(\sigma_i)$; $t \mathrel{+}= (\text{rounds used})$
    \ENDFOR
    \STATE \emph{Uplink}: every node forwards $b_i$ toward $h^\star$ once it has heard from all its children; the hub absorbs packets until a round passes with none arriving, yielding $B = \sum_i b_i$ (Prop.~\ref{prop:aggbcast}); no node needs $D$ in advance
    \STATE $(m, \mathrm{done}) \leftarrow \textsc{HubDecide}(B)$; $t \mathrel{+}= 1$
    \STATE \emph{Downlink}: hub sends $(m, D, \mathrm{done}, \mathrm{hops}{=}0)$ to $\mathrm{ch}(h^\star)$; each node that receives $(m, D, \mathrm{done}, \mathrm{hops})$ sets its own depth $d_i \leftarrow \mathrm{hops}+1$, updates $\sigma_i$ from $m$, and forwards $\mathrm{hops}{+}1$ to its children; this is how every node first learns $D$ and $d_i$
\UNTIL{$\mathrm{done}$ \textbf{or} $t \geq T$}
\end{algorithmic}
\end{algorithm}

\subsection{\merw vs. Minimum Diameter Tree}
\label{sec:mdst-gap}

\merw sets its root at $h^\star$, the highest eigenvector centrality node.
This is in general \emph{not} the graph's absolute center, i.e., the root that
would minimize tree depth and give a minimum diameter spanning tree (MDST)
\citep{hakimi1964,hassintamir1991}. Thus the tree depth $D =
\mathrm{ecc}_G(h^\star)$, which sets the $O(D)$ communication overhead of
Theorem~\ref{thm:coordinator}, can exceed $D' = R(G)$, the depth an MDST would
achieve. We bound this gap in two regimes, first an unrestricted worst case and
then two conditions, one on degree and one on spectral expansion, under which
the gap is small.

\paragraph{Worst case.} Consider a graph formed by attaching a high-degree cluster
to the end of a long path: $h^\star$ settles inside the cluster, since that is
where eigenvector centrality concentrates, while the true center sits near the
path's midpoint. As the path grows, $D$ approaches $2D'$, matching the tight
general bound $D' \geq D(G)/2$ that holds for \emph{any} choice of root
\citep{hakimi1964}; \merw's hub does no better here than an arbitrary one. This
construction has a cut vertex on the path.

The bound below needs $h^\star$ to have enough neighbors that any other node is
reachable in a couple of hops, so we first show that being the highest
eigenvector centrality node already guarantees a minimum degree, with no extra
assumption on $G$.

\begin{proposition}[Hub degree lower bound]
\label{prop:hubdeg}
The hub's degree is at least the graph's spectral radius: $\deg(h^\star) \geq \lambda$.
\end{proposition}

\begin{theorem}[Dense graphs: depth at most 2]
\label{thm:mdst-gap}
Let $\delta_{\bar N}(h^\star) = \min_{u \notin \mathcal{N}_G(h^\star)\cup\{h^\star\}} \deg(u)$ be the smallest degree among the nodes $h^\star$ is not directly connected to. If $\deg(h^\star) + \delta_{\bar N}(h^\star) \geq N$, then $D \leq 2$.
\end{theorem}

\begin{proof}
Take any $u \neq h^\star$ with $u \notin \mathcal{N}_G(h^\star)$; every neighbor of $h^\star$ already has $d(h^\star,\cdot)=1$, so it suffices to show $d(h^\star,u)\leq2$ for such $u$. Both $\mathcal{N}_G(h^\star)$ and $\mathcal{N}_G(u)$ are subsets of $V\setminus\{h^\star,u\}$, and since $\deg(u)\geq\delta_{\bar N}(h^\star)$, their sizes satisfy $|\mathcal{N}_G(h^\star)|+|\mathcal{N}_G(u)| \geq \deg(h^\star)+\delta_{\bar N}(h^\star) \geq N$ by the hypothesis. Since $|V\setminus\{h^\star,u\}|=N-2 < N$, the two neighbor sets cannot be disjoint, so some $w$ is a common neighbor, giving $d(h^\star,u)\leq2$ via $h^\star\text{-}w\text{-}u$.
\end{proof}

Since $D' \geq 1$ on any connected graph with more than one node,
Theorem~\ref{thm:mdst-gap} says \merw's tree is at most one hop deeper than the
true MDST whenever this condition holds. By Proposition~\ref{prop:hubdeg}, the
hub's degree is at least $\lambda$, so the condition is easier to satisfy the
more concentrated $G$'s spectral radius is around $h^\star$. It requires only
$h^\star$ and the nodes outside its immediate neighborhood to be
well-connected, not every node in $G$.

\paragraph{Well-connected graphs.} Theorem~\ref{thm:mdst-gap}'s hypothesis is a strong density requirement. A weaker condition, the graph's algebraic connectivity $\lambda_2$ (the second-smallest Laplacian eigenvalue) bounded away from $0$, already bounds the gap logarithmically in $N$, with no density requirement on individual nodes.

\begin{theorem}[Depth gap under bounded algebraic connectivity]
\label{thm:mdst-gap-spectral}
Let $\lambda_2$ be the algebraic connectivity of $G$ (the second-smallest eigenvalue of the graph Laplacian $L = \mathrm{Diag}(\deg) - A$) and $\lambda_\infty$ its largest Laplacian eigenvalue. Then
\[
D - D' \leq \left\lceil \frac{\lambda_\infty + \lambda_2}{4\lambda_2} \ln(N-1) \right\rceil.
\]
\end{theorem}

This bound does not depend on $h^\star$'s own degree, only on the graph's
overall expansion, so it holds much more broadly than Theorem~\ref{thm:mdst-gap},
including on sparse graphs. It grows with $N$ instead of staying at 2, and it
fails exactly on graphs like a long path or cycle, where $\lambda_2$ vanishes as
$N$ grows and the worst case from earlier is observed. Theorem~\ref{thm:mdst-gap}
already covers Erd\H{o}s--R\'{e}nyi graphs with no extra proof: for $G(N,p)$
with $p$ above $1/2$ by a constant margin, degrees concentrate tightly around
$(N-1)p$, so its hypothesis holds with probability approaching 1, and $D-D'$ is
at most one hop with high probability. \merw does not try to avoid the worst
case directly, since doing so would need the same all-pairs distances that
exact MDST construction depends on, which is the global knowledge \merw is
built to avoid.

$\lambda_2$ is also what governs the cost of the fault tolerance in
Section~\ref{sec:ft}: a lost node is repaired by a warm-started re-election
whose cost is set by the same spectral gap. When $\lambda_2$ is bounded away
from $0$, that repair is cheap; on sparse or path-like graphs, where $\lambda_2$
is small, it is not.

\subsection{Instantiation: Regret Minimization}
\label{sec:regret}

With $K$ arms of unknown means $\mu_1 \geq \cdots \geq \mu_K$ and sub-Gaussian rewards, the objective is to minimize $R(T) = \sum_{i}\sum_{t=1}^T \mathbb{E}[\mu_1 - \mu_{a_i(t)}]$. Each agent $i$ keeps two length-$K$ vectors, $n_i, s_i \in \mathbb{R}^K$, its own per-arm pull counts and reward sums since the last cycle; these are exactly the local contribution $b_i = (n_i, s_i)$ that Algorithm~\ref{alg:merw-generic} aggregates. The hub sums these into the running totals $\hat n = \sum_i n_i \in \mathbb{R}^K$ and $\hat s = \sum_i s_i \in \mathbb{R}^K$, one pull count and one reward sum per arm, pulls the UCB1-maximizing arm \citep{auer2002}, i.e., $\arg\max_k \hat s_k/\hat n_k + \sqrt{c\log(P)/\hat n_k}$, and broadcasts $(\hat n, \hat s, P)$ on the downlink. The scalar $P = \sum_k \hat n_k$ is the total pull count across all arms; it is carried alongside $\hat n$, rather than recomputed by summing $\hat n$ every round, so that every node applies the same exploration bonus without an extra reduction (following \citep{martinez2019decentralized}). Each relay cycle lasts $2D+1$ rounds: one pull round, $D$ uplink rounds, one hub-decision round, $D$ downlink rounds.

\begin{corollary}[Regret matches centralized UCB]
\label{cor:regret}
Under the \merw relay protocol, the leading term of the expected group cumulative regret matches a centralized UCB agent on $P$ total pulls, with $P = \Theta(NT/(2D+1))$.
\end{corollary}

\subsection{Instantiation: Best Arm Identification}
\label{sec:bai}

In BAI, given confidence $\delta$, the goal is to identify $a^\star =
\arg\max_k \mu_k$ with probability $\geq 1-\delta$ using as few total pulls as
possible. \merw-BAI adapts \citet{hillel2013distributed}'s Algorithm 3 to the
same generic protocol, run in elimination rounds $r = 1, 2, \ldots$ over a
surviving arm set $S_r \subseteq \{1,\ldots,K\}$, initialized to
$S_0 = \{1,\ldots,K\}$ and only shrinking. In round $r$, every agent $i$
pulls each arm $k \in S_{r-1}$ exactly $L_r = \lceil t_r - t_{r-1}\rceil$
times, where $t_r = \frac{2}{N\epsilon_r^2}\ln\frac{4Kr^2}{\delta}$ is their
elimination schedule and $\epsilon_r = 2^{-r}$ ($t_0 = 0$), and its local
contribution is the vector of its own reward sums over that round,
$b_i = (b_{i,k})_{k \in S_{r-1}}$. Every node computes $L_r$ independently
from this formula using the shared, known $N$, so no per-round negotiation is
needed, and every node computes the same $L_r$. This is unlike \merw-UCB,
whose UCB bonus depends only on the total-pull count $P$ as it is actually
observed through the downlink, never on $N$ directly: BAI needs $N$ to compute
the very first $L_1$, before any uplink has happened, while UCB never needs it
at all. The hub sums the aggregate $B_k = \sum_i b_{i,k}$ into the
empirical mean $\bar\mu_k = B_k / (N L_r)$ for each $k \in S_{r-1}$, eliminates
every arm more than $\epsilon_r$ below the best empirical mean,
$S_r = \{k \in S_{r-1} : \bar\mu_k \geq \max_{k'} \bar\mu_{k'} - \epsilon_r\}$,
and broadcasts $S_r$ on the downlink instead of a running statistic like
$(\hat n, \hat s, P)$. The hub, not any other node, determines when to stop,
once $|S_r|=1$, and this outcome is broadcast on the downlink like any other
decision. Consequently \merw-BAI's round length $L_r + 2D$ shrinks across
elimination rounds as $\epsilon_r$ tightens, unlike \merw-UCB's fixed $2D+1$.

\begin{corollary}[BAI sample complexity matches centralized BAI]
\label{cor:bai}
The total number of arm pulls required by \merw-BAI to identify $a^\star$ with
probability $\geq 1-\delta$ matches the centralized algorithm of
\citet{hillel2013distributed}; the only overhead is $2D$ extra time steps per
elimination round.
\end{corollary}

\section{Fault Tolerance}
\label{sec:ft}

A centralized system like the one identified by \merw has one obvious weakness:
if the hub fails, the system fails with it, since every agent's information
passes through that single point. More generally, the failure of any non-leaf
node can break the routing tree $\mathcal{T}$, and thus the protocol. We
describe a fault-tolerant variant, \merwft, that mitigates the effect of node
deletion for sufficiently well-connected graphs, as per
Sec.~\ref{sec:mdst-gap}. Under \merwft, any single node $v\in G$ can fail, and
the surviving nodes can efficiently recompute the eigenvector centrality and
thus rebuild a new routing tree that spans the surviving graph $G\setminus \{v\}$.

\subsection{Node Removal as a Perturbation}
\label{sec:perturbation}

Removing a node $v$ deletes row and column $v$ of the adjacency matrix:
\[
	\tilde A = A + E, \qquad E = -\bigl(A e_v e_v^\top + e_v e_v^\top A\bigr),
\]
a symmetric perturbation of rank at most $2$. Power iteration on $\tilde A$ needs no
change of procedure: a surviving node's update $\tilde w_i = \sum_{j\sim i}
w_j$ simply stops including the dead neighbor. So a warm start is natural,
survivors keep their converged $w_i$ and continue on $\tilde A$ instead of
resetting to $w_i = 1$.

By Davis--Kahan \citep{daviskahan1970}, the eigenvector's movement is bounded
by $\|E\|_2/\gamma = \sqrt{\deg(v)}/\gamma$, where $\gamma$ is $\tilde A$'s
eigenvalue gap: largest when $v$ is the hub, smallest for a low-degree node.
Reaching tolerance $\epsilon$ takes $O(\log(\sqrt{\deg(v)}/(\gamma\epsilon)))$
rounds. On an expander $\gamma$ is bounded away from $0$, so it is a constant
and the bound is $O(\log(\sqrt{N}/\epsilon))$, using $\deg(v) \leq N$; since
$\log\sqrt{N} = \tfrac12\log N$, this is the same order as
$O(\log(N/\epsilon))$, against $\tau_{\mathrm{init}}$ for a cold start. The
well-connected regime of Section~\ref{sec:mdst-gap} is where re-election is
cheap.

\begin{proposition}[Warm-started re-election]
\label{prop:reelection}
After any node failure, the warm-started power iteration on $\tilde A$
converges to the leading eigenvector of $\tilde A$ from the pre-failure
$\tilde\psi$, and reaching tolerance $\epsilon$ takes
$O\bigl(\log(\sqrt{\deg(v)}/(\gamma\epsilon))/\log(\lambda_1/\lambda_2)\bigr)$
rounds, where $\gamma$ is the eigenvalue gap of $\tilde A$. On a graph with
$\lambda_2$ bounded away from $0$, this is $O(\log(N/\epsilon))$.
\end{proposition}

\subsection{The Well-Connected Regime}
\label{sec:spectral-regime}

The condition for cheap re-election is a spectral gap bounded away from $0$,
i.e., $G$ is an expander: this is the same condition, through
$\kappa(G) \geq \lambda_2$ \citep{fiedler1973}, that keeps $G$ connected under
node removal, and the one that makes the initial hub emergence fast. Expansion,
not edge count, is what matters: a sparse random regular graph re-elects just
as cheaply as a dense one, while a dense graph with a near-bottleneck does
not, since removing a central node can then swing centrality across the
bottleneck and the warm start is a poor initializer. \merwft is therefore a
fault-tolerance result for the well-connected regime, which is the regime
where fault-tolerant coordination is normally posed.

\subsection{Failure Detection and Synchronized Restart}
\label{sec:sync}

Before the survivors can re-run the iteration they must agree, without a
coordinator, that a failure happened and start together, in step. Every node
already learned its depth $d_i$ and the tree height $D$ from the first
downlink (line~13 of Algorithm~\ref{alg:merw-generic}), where depth is just
the hop count that message carried; every ordinary cycle is one
uplink-and-downlink traversal of $2D+1$ hops, and detection reuses this
schedule directly. Each node runs one uniform timer $\tau = 2D+1$ hops, the
length of a cycle, and suspects a neighbor dead when its expected message
misses that window; the rule is identical for every node, and the hub is
detected by its children exactly as any node is detected by its parent. A dead
node severs the tree into components, each announced by its own first
detector and flooded through its intact component; the uniform timer keeps
detectors within $D$ hops of each other, and a flood spans a component in $D$
hops.

\begin{proposition}[Failure horizon]
\label{prop:horizon}
Under the uniform timer $\tau = 2D+1$, after any single node failure every
surviving node receives the failure notice within $2D$ hops of the earliest
detection.
\end{proposition}

This is what lets the survivors restart synchronously with no clock
agreement: the notice carries the same hop counter, so a node $h$ hops from
its detector waits the remaining $2D-h$ hops before its first iteration step,
and every node's travelled-plus-waited hops sum to $2D$. The depth $d_i$ each
node already holds sets the detection deadline, bounds the skew, and serves as
the restart countdown, with nothing new added to any message. The
supplementary material gives the construction and the horizon proof in full.
Once the iteration converges, the node whose refreshed $\tilde\psi_i$ is now
the global maximum becomes the new hub, and the tree is rebuilt around it by
the same max-flooding as at initialization, over $G\setminus\{v\}$, so its
height is discovered from the first ordinary cycle exactly as $D$ was at the
start.

\iffalse
\subsection{Recovery}

So far we assumed a failed node stays down. The general version of \merwft
also handles a node coming back. A recovering node $v$ has no reliable
state: it may have missed any number of promotions and re-routings while it
was down. So it does not try to recover its old position in $\mathcal{T}$;
it rejoins as if it were new. $v$ sends a \textsc{Ping} to each graph
neighbor. Any live neighbor $j$ that is already attached replies with its
current depth $d_j$, and $v$ attaches to whichever neighbor gives the
smallest $d_j$, breaking ties by smallest identity, as in the initial
max-flooding. It sets $d_v = d_j + 1$ and waits for the next downlink
through $j$ to fill in its mirrored state $(\hat n, \hat s)$, the same way
any node does at a cycle boundary.

\begin{proposition}[Recovery correctness]
\label{prop:recovery-correct}
If $v$ attaches to a live, already-attached neighbor $j$ upon recovery, the resulting structure remains a valid spanning tree over all currently live nodes, with no cycles introduced, regardless of which node currently serves as hub.
\end{proposition}

This works the same way even under repeated failure. If $v$ happens to
reconnect through a node that was hub at some earlier point, or through one
promoted since, nothing changes: recovery only needs $j$ to be live and attached,
not any particular identity for $j$.

\section{Discussion}
\label{sec:discussion}

Section~\ref{sec:mdst-gap} shows that the gap between $D$ and $D'$ is small
on dense or well-expanding graphs. A complete relation that also covers
sparser graphs and other random graph models is left for future work.

The graph must be connected and undirected, and stays that way except for
the node failures \merwft handles. $D$ can be large on sparse or path-like
graphs; this is exactly the regime Section~\ref{sec:mdst-gap} identifies.
BAI needs $N$ inside the elimination schedule of \citet{hillel2013distributed}
itself, before the very first round; regret minimization only needs $N$ to set
$\tau_{\mathrm{init}}$ and the gossip-round count during setup, never again
afterward. \merwft only covers fail-stop failures: a node works
correctly, goes silent, or comes back and rejoins through the recovery
mechanism of Section~\ref{sec:ft}. It has no mechanism for a node that stays
active but sends incorrect messages, and none for two nodes failing within the
same re-election. Both are left for future work.

\section{Conclusion}
\label{sec:conclusion}

A coordinator can emerge from purely local computation: no node ever needs
to see the graph's global structure, or even know how many other agents
there are. \merw builds this coordinator from eigenvector centrality alone,
using only local gossip and a short flooding phase, and the resulting hub
is statistically sufficient for any decision rule that depends only on
globally aggregated statistics. We instantiated this twice, for regret
minimization and for best arm identification, and both match centralized
performance up to a bounded, $O(D)$ communication overhead per round.

Any single node, hub or not, can still fail. \merwft handles this: it
removes the failure points a tree creates at zero or near-zero steady-state
cost, and it brings a returning node back into the tree without disrupting
anything else. What is left, as Section~\ref{sec:discussion} lays out, is
mostly about tightening guarantees already in place: a general bound
relating $D$ to the true minimum diameter, and a fault model that goes
beyond fail-stop failures.
